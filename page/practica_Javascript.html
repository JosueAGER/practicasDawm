Practica de Javascript

Javascript es un lenguaje de programacion que se ejecuta tanto en el frontend como en el backend

Javascript lo usamos en varios Frameworks

  1. React
  2. Vue.js
  3. Angular
  4. Svelte/SvelteKit
  5. Next.js
  6. Nuxt.js
  7. Astro

Para programar en Javascript, usted hara uso de sus conocimientos en programacion
Variables: Locales, globales, estaticas/constantes
Estructuras de control: if/else
Bucles: for/while
Funciones

Tipos de variables

  • Let: Locales - Dentro de una función o bloque de programación
  • var: Globales - Las podemos utilizar en cualquier parte del desarrollo
  • const: Son varialbes de valores fijos, que no pueden cambiar de valor durante su ejecución

Tipos de Datps

  • String: caracteres alfanumericos
  • Number: valores numericos
  • Booleano: True/False
  • Arrays: Listas["Elemento Lista #1","Elemento Lista #2", "Elemento Lista #n"]
  • Ojbect - Diccionarios: NombreDiccionario{"Clave1:" "Valor1", "Clave2": "Valor2", "ClaveN": "ValorN"}
  • Null - Valor indefinido

Programación en javascript

  • Condiciones: If/else
  • Estructuras: Switch
  • Ciclos/bucles: For, While, do-while
  • for ... of - para recorrer colecciones (arrays)
  • for ...in - para recorrer los obj/dicc (clave)

Momento - Lugar en el que se ejecuta javascript

  1. Script dentro del html - embebido en medio del html (inline Script Early)
  2. Dentro de un control - Uso de los atributos en los elementos HTML (Inline Event Handlers)
  3. Script al final (Antes de cerrar el body) - Script embebido antes de cerrar el body (Inline Script Late)
  4. Archivo Externo (.js) - Script externo vinculado al HTML - (External Script)

Eventos programables Javascript dentro de una página web

  1. DOMContentLoaded

    Este evento se dispara cuando el DOM se ha cargado completamente pero sin esperar por los CSS, imagenes y subframes.

  2. load

    El evento load se dispara cuando una página web y todos sus recursos (imágenes, hojas de estilo, scripts, etc.) se han cargado completamente.

  3. unload

    Este evento se dispara cuando la página está a punto de ser descargada , ya sea porque el usuario navega a otra página o cierra la ventana del navegador

  4. beforeunload

    Este evento se activa antes de que la página sea descargada, permitiendo que el navegador muestre un mensaje de advertencia al usuario si está a punto de abandonar la página.

  5. visibilitychange

    Este evento se activa cuando la visibilidad de la página cambia, es decir, cuando el usuario minimiza, cambia de pestaña o se aleja de la página.

  6. readystatechange

    Este evento se dispara cada vez que cambia el estado de carga de un documento.

  7. fullscreenchange y fullscreenerror

    fullscreenchange Se activa cuando un elemento entra o sale del modo de pantalla completa.

    fullscreenerror Se activa si hay un error al intentar poner un elemento en modo de pantalla completa Ambos eventos son útiles para gestionar el modo de pantalla completa en la página, permitiendo, por ejemplo, mostrar mensajes de error o actualizar la interfaz cuando el usuario entra o sale del modo de pantalla completa.

  8. copy, cut, paste

    Estos eventos se disparan cuando se realiza una acción de copiar (copy), cortar (cut) o pegar (paste) en un área seleccionable de la página.

  9. selectionchange

    Este evento se dispara cuando el rango de selección de texto en el documento cambia (por ejemplo, cuando el usuario selecciona o deselecciona texto).

  10. scroll

    Permite realizar acciones basadas en el desplazamiento, como cargar contenido adicional cuando el usuario llega al final de la página o mostrar/ocultar elementos.
